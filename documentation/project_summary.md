# Banking Fraud Detection System Documentation

## Project Objective

The objective of this project is to identify suspicious banking transactions using data analytics and visualization techniques.

## Workflow

1. Generate synthetic banking transaction data using Python.
2. Clean and preprocess the raw transaction dataset.
3. Analyze fraud patterns using SQL queries.
4. Visualize insights through a Streamlit dashboard.
5. Document results and findings.

## Key Results

* Total Transactions: 1000
* Fraud Transactions: 57
* Safe Transactions: 943
* UPI showed higher fraud occurrences compared to other payment methods.
* Net Banking showed the lowest fraud count.

## Dashboard Components

### Dashboard Overview

Displays total transactions, fraud transactions, safe transactions, and transaction preview.

### Fraud vs Safe Transactions

Compares legitimate and fraudulent transactions.

### Payment Type Distribution

Shows transaction distribution across payment methods.

### Excel Analysis

Provides fraud analysis using pivot tables and charts.
